Model Gateway İhtiyaç Çerçevesi Laboratuvarı

Kitap bağlantısı: Cilt 1, Bölüm 17 — 17.5 (KAVRAM — Model Gateway). Kendi kurumunuzun mevcut durumunu aşağıda kurun; laboratuvar yalnızca kaç sağlayıcı kullandığınıza değil, maliyet izleme/yönlendirme/yedekleme boşluklarına bakarak "gateway'e ihtiyacınız var mı" sorusuna kanıta dayalı bir cevap üretir. Kurulum gerekmez, sayfa tek başına çalışır. Hiçbir model, araç veya sağlayıcı adı geçmez — bu laboratuvar isimden bağımsız bir çerçevedir.

2

Bu değerlendirme nasıl kuruluyor?

Sağlayıcı sayısı ikiden azsa, gateway'in çözdüğü temel problem (birden fazla sağlayıcıyı tek noktadan karşılaştırma/yönlendirme) henüz ortaya çıkmamış demektir; doğrudan sağlayıcı API'si yeterli kabul edilir. İki veya daha fazla sağlayıcı varsa, laboratuvar dört noktayı (maliyet izleme, yönlendirme, yedekleme, göreve göre elle geçiş) tarar; eksik olan her nokta bir sinyal sayılır. Üç veya daha fazla sinyal birikmişse gateway önerilir; bir-iki sinyal varsa "değerlendirilebilir ama henüz zorunlu değil" sonucu döner; hiç sinyal yoksa (yani birden fazla sağlayıcı olsa bile takip zaten elde tutuluyorsa) doğrudan API yeterli kabul edilir.

Bu, kitaptaki KAVRAM kutusunun ("gateway kullanmak maliyeti otomatik olarak düşürmez; ara katmanın kendi gecikmesi olabilir") uyarısıyla tutarlıdır: sayaç yalnızca sağlayıcı sayısını değil, o sayının yönetiminden doğan somut boşlukları ölçer.

Bu laboratuvar model, araç, sağlayıcı adı veya fiyat/benchmark iddiası üretmez; yalnızca 17.5'teki çerçeveyi uygular. Aynı laboratuvarın test edilmiş Python sürümü model_gateway_framework.py dosyasındadır; çalıştırma adımları için laboratuvarın README sayfasına, öteki laboratuvarlar için Kitap Destek Deposu ana sayfasına bakabilirsiniz.
